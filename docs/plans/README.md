# Development Plans

This directory holds historical design and implementation plans. They are useful
for rationale, call-site checklists, and verification notes, but they are not the
primary source of truth.

When a plan conflicts with the implementation or `DEVLOG.md`, prefer the code
and `DEVLOG.md`.

Plans are plans: they track scope, steps, progress, and mechanical verification.
**Experimental results and their interpretation belong in the relevant DEVLOG**
(root `DEVLOG.md`, or an extension's local one), not in plan trackers — a plan
entry should at most say a run happened and point to the DEVLOG entry. (Recorded
2026-06-12; earlier plans such as `PLAN-arbuffer.md` predate this rule and carry
result numbers in their trackers.)
